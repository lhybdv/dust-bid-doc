# 副标模型方案：沙尘影响风险评估与场景服务模型

## 一、模型定位

本项目中的模型建设不以沙尘核心预测模型创新为主要目标，不替代项目主标中沙尘气象人工智能预测模型的研发任务，而是围绕示范应用平台建设需求，构建面向公众服务和行业应用的沙尘影响风险评估与场景服务模型。

该模型主要用于承接上游沙尘预测模型输出结果、实时监测数据、业务阈值规则和外部智能体服务结果，将专业气象预测信息转化为面向交通、人体健康、旅游和公众服务的风险等级、影响提示和服务建议。

因此，本项目模型属于应用侧模型、场景侧模型和服务转化模型，重点解决“预测结果如何被业务使用、如何被公众理解、如何支撑行业服务”的问题，而不是重复开展沙尘核心预测模型研发。

## 二、总体思路

本项目拟采用“主模型结果接入 + 实况监测订正 + 场景风险分级 + 图关系辅助分析 + 智能体服务融合”的技术路线，构建沙尘影响风险评估与场景服务模型。

模型以沙尘预测结果、能见度、PM10、AQI、风速风向、区域位置、道路路段、景区点位、人口敏感区域等信息为输入，通过规则模型、轻量机器学习模型和图关系分析方法，对不同服务对象受到沙尘天气影响的程度进行评估。

模型输出不直接强调沙尘浓度预测本身，而是重点输出：

- 公众风险等级；
- 道路通行影响等级；
- 健康防护建议等级；
- 旅游出行适宜性等级；
- 重点区域影响提示；
- 智能体服务调用参数；
- Web 端和微信小程序端展示内容。

通过该方式，既可以充分承接主标模型成果，又能够体现本标段在平台应用、场景转化和用户服务方面的建设价值。

## 三、模型技术路线

### 3.1 上游预测结果接入

平台首先接入上游沙尘预测模型输出结果，包括未来不同时效的沙尘影响范围、强度等级、浓度变化趋势、风险区域和置信度等信息。

接入后，平台不对主预测模型进行重复训练，而是对预测结果进行服务化处理，包括格式转换、空间裁剪、时间聚合、区域匹配、等级映射和接口封装，使其能够被 Web 端、微信小程序端、行业接口和智能体服务调用。

### 3.2 实况监测融合订正

为提升应用侧服务的可靠性，平台将融合实时监测数据对预测结果进行动态订正。可接入的数据包括：

- 能见度实况；
- PM10、AQI 等大气环境数据；
- 地面风速、风向、湿度等气象观测数据；
- 卫星遥感沙尘识别结果；
- 重点区域实况反馈数据。

通过实况数据与预测结果的对比，平台可对当前风险等级、影响范围和服务提示进行动态调整，避免只依赖单一预测结果造成服务偏差。

### 3.3 场景风险分级模型

针对不同应用场景，构建差异化风险分级模型。该模型可采用“业务规则 + 阈值分级 + 轻量机器学习”的组合方式实现。

交通场景主要关注能见度、风速、沙尘强度、道路等级和影响时段，将其转化为道路通行风险等级、限速建议和重点路段提示。

人体健康场景主要关注 PM10、AQI、沙尘持续时间和人群敏感性，将其转化为健康影响等级、敏感人群防护等级和公众防护建议。

旅游场景主要关注景区位置、沙尘影响范围、能见度、风力、影响时段和户外活动类型，将其转化为景区影响等级、出行适宜性等级和行程调整建议。

公众服务场景主要关注用户位置、关注区域、未来影响趋势和防护建议，将专业结果转化为简明易懂的风险提示和防护提醒。

### 3.4 图关系辅助分析模型

为增强不同区域、站点和服务对象之间的空间关联表达能力，平台可引入轻量化图关系分析方法。该部分不作为主预测模型，而是作为应用侧的辅助分析模型。

可将沙源地、气象站点、空间网格、道路路段、景区、城市区域和重点人群服务区抽象为图节点，将地理邻近关系、主导风向关系、历史沙尘影响关系和业务服务关系抽象为图边，构建沙尘影响服务关系图。

图关系模型主要用于：

- 分析沙尘影响从上游区域向下游区域的传播关系；
- 将预测结果匹配到道路、景区、城市和重点服务区域；
- 支持相邻区域风险联动提示；
- 辅助判断重点区域是否需要提前推送服务信息；
- 为智能体服务提供结构化上下文输入。

该方法能够体现一定技术深度，但不与主标中的核心预测模型形成竞争关系，更适合作为平台应用侧的空间关系增强能力。

### 3.5 智能体服务融合

平台接入已有或外部提供的交通、人体健康、旅游等智能体服务。风险评估模型将预测结果、实况数据、区域信息、用户场景和风险等级整理为结构化输入，提交给智能体服务进行分析。

智能体返回的分析结论、服务建议和提示内容，经平台解析后展示在 Web 端和微信小程序端。平台负责完成接口适配、参数组织、结果解析、异常处理、日志记录和内容展示，不将智能体底层能力作为本项目自研内容。

## 四、模型结构设计

本项目模型可分为五个层次：

### 4.1 数据输入层

输入内容包括：

- 上游沙尘预测模型结果；
- 实况气象监测数据；
- 大气环境监测数据；
- 卫星遥感识别产品；
- 道路、景区、行政区划等空间数据；
- 业务阈值和服务规则；
- 用户位置或关注区域信息。

### 4.2 结果处理层

对输入数据进行统一处理，包括：

- 数据格式转换；
- 空间匹配；
- 时间聚合；
- 区域裁剪；
- 等级映射；
- 异常值处理；
- 结果缓存。

### 4.3 风险评估层

按照不同应用场景进行风险评估，包括：

- 公众风险评估；
- 交通通行风险评估；
- 人体健康影响评估；
- 旅游出行影响评估；
- 重点区域风险联动评估。

### 4.4 智能体融合层

将风险评估结果与场景上下文组织为智能体输入参数，调用外部智能体服务，获取更贴近业务表达的分析结论和服务建议。

### 4.5 服务输出层

输出内容包括：

- Web 端专业用户展示内容；
- 微信小程序公众服务内容；
- 行业系统 API 接口数据；
- 预警提示和消息推送内容；
- 智能体分析结果；
- 服务日志和运行记录。

## 五、推荐模型名称

建议在标书中使用以下名称之一：

### 方案一：稳妥正式型

沙尘影响风险评估模型

### 方案二：更贴近示范应用型

沙尘监测预警场景服务模型

### 方案三：体现智能化但不过度抢主标型

沙尘影响智能评估与服务转化模型

### 方案四：适合写技术路线型

沙尘多场景风险分级与服务推荐模型

综合建议使用：

沙尘影响智能评估与服务转化模型

这个名称比较好，因为它强调的是“影响评估”和“服务转化”，不是“沙尘预测核心模型”，可以自然避开主标模型研发内容。

## 六、可写入标书的正式表述

本项目不重复建设沙尘气象核心预测模型，而是围绕沙尘监测预警示范服务应用需求，构建沙尘影响智能评估与服务转化模型。该模型面向 Web 端、微信小程序端和行业服务接口，重点承接上游沙尘预测模型输出、实时监测数据和外部智能体服务能力，将专业气象预测结果转化为公众易理解、行业可使用、平台可展示的风险等级、影响提示和服务建议。

模型采用“预测结果接入、实况监测订正、场景风险分级、图关系辅助分析、智能体服务融合”的技术路线。通过接入沙尘预测结果、能见度、PM10、AQI、风速风向、道路路段、景区点位、重点人群服务区等数据，构建面向交通、人体健康、旅游和公众服务的风险评估规则与分级模型。对于交通场景，形成道路通行风险等级、能见度影响提示和限速建议；对于人体健康场景，形成敏感人群防护等级和健康风险提示；对于旅游场景，形成景区影响等级、行程调整建议和应急避险提示；对于公众服务场景，形成当前位置风险提醒、预警提示和个性化防护建议。

为增强空间关联分析能力，模型引入轻量化图关系分析方法，将沙源地、气象站点、空间网格、道路路段、景区和重点服务区域抽象为图节点，将地理邻近关系、主导风向关系、历史影响关系和业务服务关系抽象为图边，辅助开展影响范围匹配、重点区域识别和服务对象关联分析。该图关系模型作为应用侧辅助分析能力，不替代上游沙尘核心预测模型，主要用于提升平台服务结果的空间一致性和场景解释能力。

模型评估结果和场景上下文将作为智能体服务调用输入，平台通过标准化接口接入交通、人体健康、旅游等智能体服务，对智能体返回的分析结论、风险说明和服务建议进行解析、审核、结构化展示和消息推送。通过该方式，平台能够将上游模型预测能力、实时监测能力和智能体服务能力进行统一集成，形成面向公众和专业用户的沙尘监测预警示范服务能力。
