# 重写标书用户 brief v2

## 一、文件用途

本 brief 用于重新撰写《宁夏沙尘预测AI模型训练及示范应用项目》投标文件。`work/40_drafts/v1.md` 不再作为正文底稿继续扩写，但其中的商务响应、实施安排、平台功能、运维培训等内容可作为素材库复用。

重写目标不是在旧稿上继续补丁式修改，而是以采购需求和用户真实定位为准，重新组织一份更稳、更聚焦、更像投标文件的正文。

## 二、总体定位

本项目应写成一套独立完整的投标方案，不能表述为依赖“主标”或其他竞争方输出结果后再开展建设。采购需求中明确包含沙尘预测AI模型训练和示范应用平台研发，因此新稿必须完整响应模型训练、平台研发、微信小程序、行业API、三领域智能体接入、培训和售后服务等内容。

同时，技术表达的重心不应只堆砌“核心预测模型创新”。本方案的优势应放在沙尘预测能力与应用场景之间的转化能力，即将沙尘预测、实况监测、业务阈值和智能体服务组织成可使用、可展示、可联动的行业服务。

推荐核心表述：

> 本项目以沙尘动态预测AI模型训练为基础，以沙尘监测预警示范服务场景应用平台为载体，构建覆盖公众、交通、人体健康、旅游等场景的沙尘影响智能评估与服务转化能力。方案既响应采购文件关于预测提前量、准确度和平台性能的硬性要求，也突出预测结果向风险等级、行业提示、公众建议和联动接口转化的应用价值。

## 三、必须修正 v1 的问题

1. 不能再把全文写成“素材堆叠型长文”。新稿需要围绕评分和采购响应组织，每章有明确任务，不用为了页数反复扩写。
2. 不能写成依赖主标模型输出的从属方案。可写“接入外部数据源、数值模式、观测和遥感产品”，但不要写“等待主标提供预测结果”。

4. 智能体来源按已确认口径处理：交通、人体健康、旅游三领域智能体由宁夏移动负责建设，本项目通过标准接口接入和调用，不写成我方自研三类智能体。
5. 图关系辅助分析模型按已确认口径处理：可写自研，但定位为应用侧辅助分析能力，服务于空间关联、重点区域识别和场景解释，不要夸大为替代核心预测模型。
6. 压缩空表、占位表和模板话。商务材料可保留框架，但正文重点放在可评分、可验证、可落地的内容。

## 五、建议技术主线

新稿建议采用“模型训练 + 监测预警平台 + 场景服务转化”的三层主线。

### 1. 沙尘动态预测AI模型训练

响应采购文件对沙尘预测AI模型训练的要求，说明基于历史沙尘天气过程、气象观测、数值模式、卫星遥感、地表状况、大气环境监测等数据，构建沙尘天气历史个例人工智能训练数据集，并训练物理机理与深度学习融合的沙尘动态预测模型。

重点写：

- 多模态数据融合处理；
- 起沙、传输、沉降过程的特征构建；
- 宁夏腾格里沙漠南缘局地起沙特征适配；
- 迁移学习、少样本增强和域自适应；
- 输出未来24小时以上沙尘浓度或风险空间分布；
- 通过历史个例回溯、实况监测和卫星遥感判识开展检验评估。

注意：这部分必须满足采购指标，但不要写成泛泛的“大模型炫技”。要把模型训练和宁夏业务场景挂钩。

### 2. 沙尘影响智能评估与服务转化模型

这是本方案的差异化重点。模型训练给出预测结果后，需要进一步转化为业务人员、公众和行业系统能使用的信息。

建议写成以下能力：

- 预测结果服务化处理：格式转换、时间聚合、空间裁剪、行政区和重点区域匹配；
- 实况监测订正：融合能见度、PM10、AQI、风速风向、卫星遥感识别结果，对当前风险等级和服务提示进行动态修正；
- 场景风险分级：面向交通、人体健康、旅游和公众服务，形成差异化风险等级、提示和建议；
- 图关系辅助分析：自研轻量化空间关系分析模型，将沙源地、站点、空间网格、道路、景区、行政区等抽象为图节点，辅助识别空间传导关系和重点影响区域；
- 服务转化输出：为Web端、微信小程序、行业API和智能体调用提供结构化结果。

推荐模型名称：

> 沙尘影响智能评估与服务转化模型

### 3. 示范服务场景应用平台

平台是核心交付物之一，面向专业用户、公众用户和行业用户提供统一服务。建议按以下模块组织：

- 沙尘实况监测：气象站、环境监测、卫星遥感、能见度、PM10/AQI、重点区域监测；
- 沙尘预报预警：模型推理结果展示、风险等级分布、预警制作、审核发布、产品回溯；
- 场景服务分析：交通通行风险、健康防护建议、旅游出行适宜性、公众位置提醒；
- 智能体服务集成：通过API网关接入宁夏移动建设的三领域智能体，完成参数组织、调用、结果解析、日志记录和融合展示；
- GIS空间分析：底图、专题图、空间查询、圈选统计、等值线/等值面分析；
- 系统管理：用户权限、阈值配置、服务规则、接口调用日志、运行监控。

### 4. 微信小程序与行业API

微信小程序面向公众，突出“看得懂、用得上、能提醒”：

- 实时能见度地图；
- 当前位置沙尘风险等级；
- 未来趋势和预警提醒；
- 个性化防护建议；
- 关注区域订阅和消息推送。

行业API面向交通、文旅、健康等行业系统，突出标准化、可联动：

- 区域风险查询；
- 道路通行风险查询；
- 景区影响等级查询；
- 预警信息订阅；
- 智能体分析结果回传；
- API Key/JWT鉴权、限流、日志审计和异常处理。

## 九、成稿风格要求

新稿应当像“可信的实施方案”，而不是“AI生成的技术百科”。写作时优先使用具体动作、输入输出、验收指标和交付物，减少“全面提升、智能赋能、构建体系、深度融合”等空泛表达。
