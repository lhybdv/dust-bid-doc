# 2 总体技术方案

## 2.2 技术架构与技术路线

### 2.2.1 技术架构选型

本项目以微服务架构、流批一体化框架和B/S架构为核心，构建覆盖模型训练、平台服务和场景输出的完整技术底座。三种架构模式相互配合，支撑沙尘预测AI模型从训练到推理的全链路能力。

**微服务架构**按业务边界将系统拆分为独立服务单元，各服务独立部署、独立扩展，通过服务注册与发现机制实现协调。本项目将预测服务、实况监测服务、预警服务、场景服务、GIS服务、接口服务等拆分为独立微服务，各服务使用容器化部署，支撑动态扩缩容，响应REQ-S4-01（微服务架构）。

**流批一体化框架**将实时数据处理与离线批处理统一管理。气象站、监测站、能见度等实时数据通过流式管道处理，模型推理的批量预测和历史回溯通过批处理管道执行，两套管道共享特征存储和模型存储层，统一由流批一体处理引擎调度，响应REQ-S4-02（流批一体化框架）。

**B/S架构**采用浏览器/服务器模式，用户通过Web管理端和微信小程序访问平台服务。服务端提供JSON over HTTPS标准API，支撑跨平台、跨终端的灵活接入，响应REQ-S4-03（B/S架构）。

### 2.2.2 技术路线设计

本项目技术路线围绕"物理机理+深度学习融合"这一核心方法展开，分为数据融合处理路线、模型训练路线和平台服务路线三条主线。

**数据融合处理路线**：多源数据经质量控制后进入融合处理管道。气象观测数据（温度、湿度、风速、风向、气压）作为基础输入，数值模式产品（ECMWF、GFS）提供大尺度背景场，卫星遥感数据（风云FYD、MODIS）提供空间覆盖，地表和大气监测数据补充地面实况。各源数据经标准化、质量控制、时间对齐和空间插值后，在特征层面进行融合，生成模型输入特征张量。融合过程采用时空插值和特征交叉方法，保留各源数据的优势信息。

**模型训练路线**：融合特征进入沙尘动态预测AI模型训练管道。本模型采用物理机理引导的深度学习架构，底层网络为时序神经网络（长短期记忆网络LSTM或时序卷积网络TCN），在输入层和损失函数中嵌入沙尘传输的物理约束（如风场驱动、扩散方程）。训练过程使用历史个例数据（宁夏及周边地区历史沙尘个例）进行监督学习，采用迁移学习方法利用少量样本完成模型适配。训练完成后模型以服务化方式部署，接收实时融合特征并输出未来24小时以上沙尘风险空间分布（网格分辨率待确认）和浓度等级。

**平台服务路线**：模型推理结果推送到平台服务层后，分三路处理——实时预警判断（流式管道）、历史回溯分析（批式管道）和空间制图（GIS管道）。预警判断根据风险阈值生成预警信号并触发发布流程；历史回溯支撑模型效果评估和个例复盘；空间制图生成可视化产品供Web端和小程序展示。三路结果统一写入结果存储层，支撑多渠道调用。

### 2.2.3 技术栈组合

本项目主要技术栈选型如下：

| 技术层级 | 选型 | 说明 |
|---------|------|------|
| 基础设施 | 容器化部署（Docker/K8s） | 服务容器化，支撑弹性扩缩容 |
| 数据存储 | 时序数据库+关系数据库+对象存储 | 分别存储监测数据、业务数据、模型文件 |
| 流处理 | Flink 流批一体引擎 | 实时数据流处理和批量历史分析统一调度 |
| 模型训练 | PyTorch/TensorFlow | 深度学习框架，支撑LSTM/TCN模型 |
| 推理服务 | TensorRT/ONNX Runtime | 模型推理加速 |
| GIS引擎 | GeoServer + PostGIS | 空间数据管理和地图服务 |
| API网关 | Kong/Nginx | 统一鉴权、限流、日志 |
| 前端框架 | Vue3/React | Web管理端前端框架 |
| 小程序 | 微信小程序 | 移动端用户入口 |

技术栈选型遵循以下原则：主流开源优先，避免绑定单一厂商；社区活跃度高，文档完善；与现有气象业务系统兼容性良好，宁夏气象局已有类似技术栈（MANUAL: 确认现有系统技术栈以确保兼容）。

### 2.2.4 技术路线阶段规划

本项目技术路线分为四个实施阶段：

**第一阶段（合同签订后1-3个月）**：完成技术架构详细设计、数据接口规范制定、基础环境搭建和核心模型初步训练。关键里程碑为技术架构设计说明书和数据接口规范文档。

**第二阶段（合同签订后4-9个月）**：完成多模态数据融合管道建设、沙尘动态预测AI模型训练迭代、平台核心功能开发和首批历史个例验证。关键里程碑为模型训练报告（准确度验证）和平台功能演示。

**第三阶段（合同签订后10-18个月）**：完成场景服务接入、行业API对接、微信小程序上线和三领域智能体接口适配。关键里程碑为平台联调报告和用户验收测试。

**第四阶段（合同签订后19-24个月）**：完成全系统优化、正式运行验收、培训交付和运维体系建设。关键里程碑为最终验收报告和运维交接文档。

### 2.2.5 关键技术难点与应对

本项目技术路线面临以下关键难点：

**物理机理与深度学习融合**：单一深度学习模型在沙尘预报中易出现过拟合和泛化能力不足问题。本项目采用物理机理引导的损失函数设计，在常规交叉熵或均方误差损失基础上增加风场驱动约束和连续性约束，使模型在物理上更合理。应对措施为在模型训练前开展宁夏历史沙尘个例的物理规律分析，并据此设计约束项。

**小样本模型训练**：宁夏地区沙尘天气个例有限，纯数据驱动方法面临样本不足问题。本项目采用迁移学习方法，利用已有全国或更大范围的沙尘模型权重进行微调（fine-tuning），结合数据增强（时间序列增强、空间扰动）扩大训练样本集。

**流批一体调度复杂性**：实时预警和离线分析共用同一套处理引擎，增加了调度复杂性。本项目采用Flink作为统一引擎，通过作业链（Job Graph）管理流式和批式任务的依赖关系，设置独立的作业监控和异常告警。
