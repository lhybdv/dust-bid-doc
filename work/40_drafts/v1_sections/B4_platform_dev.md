# 沙尘监测预警示范服务场景应用平台研发

## 一、平台概述

### 1.1 平台定位

沙尘监测预警示范服务场景应用平台是本项目的核心交付物之一，面向宁夏区域提供沙尘天气实时监测、预报预警、影响评估和行业应用服务。平台采用微服务架构与流批一体化框架，支持多端接入（Web端、微信小程序端、行业API接口），通过标准化接口接入交通、人体健康、旅游等智能体服务，形成覆盖专业用户、公众用户和行业用户的综合沙尘服务能力。

### 1.2 建设目标

- 构建实时沙尘监测能力，融合卫星遥感、地面气象站、环境监测站等多源数据
- 部署沙尘动态预测AI模型，提供未来24小时以上精细化沙尘浓度空间分布预测
- 针对交通、人体健康、旅游三个示范应用领域以及公众服务需求，提供差异化场景服务
- 实现与行业管理系统的联动预警，提供标准化API接口支撑行业沙尘自动预警
- 通过微信小程序面向公众提供实时能见度地图和个性化防护建议

## 二、系统架构设计

### 2.1 总体架构

平台采用微服务架构，将系统拆分为多个松耦合的服务组件，支持独立开发、部署和扩展。

| 层级 | 服务组件 | 主要职责 |
|------|---------|---------|
| 接入层 | Web管理端服务 | 面向专业用户的Web端界面 |
| 接入层 | 微信小程序服务 | 面向公众的小程序服务端 |
| 接入层 | API网关服务 | 行业接口统一接入、流量管控 |
| 服务层 | 监测数据服务 | 实时数据接入、处理、存储 |
| 服务层 | 预测模型服务 | 模型推理、结果输出 |
| 服务层 | 场景分级服务 | 交通/健康/旅游场景风险分级 |
| 服务层 | 预警发布服务 | 预警信息生成、审核、发布 |
| 服务层 | 智能体接入服务 | 智能体调用、结果解析 |
| 服务层 | 地图服务 | GIS空间分析、地图渲染 |
| 数据层 | 时序数据库 | 气象、环境监测数据存储 |
| 数据层 | 空间数据库 | 矢量/栅格地理数据存储 |
| 数据层 | 对象存储 | 模型文件、遥感产品存储 |
| 数据层 | 消息队列 | 服务间异步消息、事件驱动 |

### 2.2 技术选型

| 技术组件 | 选型方案 | 选型理由 |
|---------|---------|---------|
| 容器编排 | Kubernetes | 成熟的微服务编排能力，弹性扩展 |
| 服务治理 | Spring Cloud Alibaba | 熔断、限流、网关一体化 |
| 数据库 | PostgreSQL + PostGIS | 关系型+空间数据一体化 |
| 时序数据 | TimescaleDB / InfluxDB | 高效时序数据存储查询 |
| 消息队列 | Kafka | 高吞吐、持久化、事件流处理 |
| 缓存 | Redis | 高性能缓存，热点数据加速 |
| 搜索引擎 | Elasticsearch | 日志检索、运维监控 |
| GIS引擎 | GeoServer + OpenLayers | 开源GIS解决方案 |

### 2.3 部署架构

平台采用容器化部署，支持在宁夏气象局私有云环境或政务云环境中部署。

- **开发测试环境**：单机Docker Compose部署
- **预生产环境**：3节点Kubernetes集群
- **生产环境**：5节点以上Kubernetes高可用集群

## 三、核心功能模块

### 3.1 沙尘实况监测模块

**功能描述：** 实时接入和处理多源监测数据，展示当前沙尘影响状态。

**数据来源：**
- 地面气象站：风速、风向、温度、湿度、气压、能见度、天气现象
- 环境监测站：PM10、PM2.5、AQI
- 卫星遥感：风云卫星沙尘产品、AOD、多通道亮温

**展示内容：**
- 全区能见度分布图（填色图）
- PM10/PM2.5浓度分布图
- 卫星云图与沙尘判识叠加
- 重点区域（沙源地、交通干线、旅游景区）实时监测
- 监测站点列表与实时数据表格

**性能指标：** 数据延迟<=5分钟，界面响应<=3秒

### 3.2 沙尘预报预警模块

**功能描述：** 展示沙尘动态预测AI模型的预测结果，支持预警制作和发布。

**展示内容：**
- 未来0-48小时沙尘浓度空间分布动画
- 未来0-72小时沙尘风险等级分布
- 重点区域预报产品（文字+图形）
- 预警信号制作工具（等级、区域、时效）
- 预警信息审核与发布流程

**模型集成：**
- 模型推理服务RESTful API调用
- 模型输出结果缓存与回溯
- 模型效果统计与监测

### 3.3 场景服务分析模块

**3.3.1 交通场景服务**

- 全区主要道路通行风险等级分布图
- 重点路段（高速公路、国省干道）风险实时监控
- 限速建议和通行提示信息
- 交通管理联动预警接口

**3.3.2 人体健康场景服务**

- 全区空气质量健康影响等级分布
- 敏感人群（老人、儿童、呼吸道疾病患者）专项提示
- 分时段健康风险预报
- 防护建议和科普知识推送

**3.3.3 旅游场景服务**

- 全区主要景区沙尘影响等级
- 出行适宜性分析和建议
- 行程调整和应急避险提示
- 文旅平台联动接口

### 3.4 智能体服务集成模块

**接入架构：**
- API网关统一接入智能体服务
- 智能体调用参数自动组织
- 智能体返回结果解析和标准化
- 智能体调用日志和计费记录

**三领域智能体接入：**

| 智能体 | 接入方式 | 服务内容 |
|--------|---------|---------|
| 交通智能体 | HTTP API | 能见度-通行限速关联分析 |
| 健康智能体 | HTTP API | 敏感人群健康防护建议 |
| 旅游智能体 | HTTP API | 行程调整和避险路线规划 |

### 3.5 地理信息系统（GIS）模块

**功能描述：** 提供空间数据管理和分析能力，支撑沙尘监测和影响评估。

**主要功能：**
- 底图服务（天地图、高清影像）
- 沙尘监测/预报专题图制作
- 空间查询和统计分析
- 圈选分析（框选区域查看统计）
- 等值线/等值面分析

### 3.6 系统管理模块

**功能描述：** 提供平台运维和管理能力。

**主要功能：**
- 用户权限管理（角色：管理员、预报员、公众用户）
- 阈值配置管理（支持根据区域、场景动态配置）
- 服务规则管理（预警发布规则、推送规则）
- 接口调用日志和监控
- 系统运行状态监控

## 四、微信小程序研发

### 4.1 小程序定位

微信小程序面向公众用户，提供便捷的沙尘监测预警服务，是平台公众服务能力的重要载体。

### 4.2 主要功能

| 功能模块 | 功能描述 |
|---------|---------|
| 当前位置监测 | 获取用户当前位置，展示当前沙尘影响状态 |
| 关注区域管理 | 用户可添加关注的城市/区县，获得定向推送 |
| 预报预警 | 未来24-72小时沙尘预报和预警信息 |
| 趋势分析 | 沙尘浓度变化趋势图 |
| 健康建议 | 基于当前风险等级的个性化防护建议 |
| 科普知识 | 沙尘天气成因、防护方法等科普内容 |
| 消息订阅 | 预警消息订阅和推送 |

### 4.3 技术实现

- **框架**：微信小程序原生框架或Taro跨端框架
- **后端通信**：HTTPS API调用平台服务端
- **地图**：微信小程序 map 组件 + 腾讯位置服务
- **缓存**：本地缓存热点数据，离线查看
- **性能优化**：分包加载，按需加载非关键模块

## 五、行业API接口

### 5.1 接口概述

平台提供标准化API接口，面向交通、文旅、健康等行业应用提供沙尘预警联动服务。

### 5.2 主要接口

| 接口名称 | 请求方式 | 主要参数 | 返回内容 |
|---------|---------|---------|---------|
| 区域沙尘风险查询 | GET | 区域编码、时间范围 | 风险等级、浓度、趋势 |
| 道路通行风险查询 | GET | 道路编号 | 通行风险等级、限速建议 |
| 景区影响等级查询 | GET | 景区编号 | 影响等级、出行建议 |
| 预警信息订阅 | POST | 区域、等级、联系方式 | 订阅ID、确认状态 |
| 智能体分析结果 | POST | 场景类型、参数 | 智能体返回的分析和建议 |

### 5.3 接口规格

- **协议**：HTTPS
- **认证**：API Key / JWT Token
- **限流**：默认100次/分钟，可申请提升
- **响应时间**：<=2秒（95分位）
- **可用性**：>=99.9%

### 7.2 数据库设计

**表7-2 数据库选型与用途**

| 数据库类型 | 选型产品 | 用途场景 |
|---------|---------|---------|
| 关系型数据库 | PostgreSQL 15 + PostGIS | 用户数据、权限数据、系统配置 |
| 时序数据库 | TimescaleDB | 气象监测时序数据（亿级记录/天） |
| 缓存数据库 | Redis Cluster | 会话缓存、热点数据 |
| 消息队列 | Apache Kafka | 实时数据流、服务间异步通信 |
| 对象存储 | MinIO | 模型文件、遥感产品存储 |

**核心数据表结构：**

```sql
-- 监测站点元数据表
CREATE TABLE m_station (
    id BIGSERIAL PRIMARY KEY,
    station_code VARCHAR(20) UNIQUE NOT NULL,
    station_name VARCHAR(100) NOT NULL,
    station_type VARCHAR(20) NOT NULL,
    latitude DECIMAL(10,6) NOT NULL,
    longitude DECIMAL(10,6) NOT NULL,
    altitude DECIMAL(8,2),
    region_code VARCHAR(12),
    status VARCHAR(10) DEFAULT 'active'
);

-- 实时监测数据时序表（TimescaleDB超表）
CREATE TABLE t_monitor_realtime (
    time TIMESTAMPTZ NOT NULL,
    station_code VARCHAR(20),
    element_code VARCHAR(20),
    value DECIMAL(12,4),
    quality_flag VARCHAR(10),
    PRIMARY KEY (time, station_code, element_code)
);
SELECT create_hypertable('t_monitor_realtime', 'time');

-- 预报结果表
CREATE TABLE t_forecast_result (
    id BIGSERIAL PRIMARY KEY,
    forecast_time TIMESTAMPTZ NOT NULL,
    valid_time TIMESTAMPTZ NOT NULL,
    region_code VARCHAR(20),
    element_code VARCHAR(20),
    value DECIMAL(12,4),
    confidence DECIMAL(5,4)
);

-- 预警信息表
CREATE TABLE t_alert_info (
    id BIGSERIAL PRIMARY KEY,
    alert_id VARCHAR(50) UNIQUE NOT NULL,
    alert_type VARCHAR(20) NOT NULL,
    alert_level VARCHAR(10) NOT NULL,
    region_code VARCHAR(20) NOT NULL,
    valid_time_start TIMESTAMPTZ NOT NULL,
    valid_time_end TIMESTAMPTZ NOT NULL,
    title VARCHAR(200),
    content TEXT,
    publish_time TIMESTAMPTZ,
    status VARCHAR(20) DEFAULT 'active'
);
```

## 九、私有云部署方案

### 9.1 集群规划

| 节点类型 | 数量 | 资源配置 | 用途 |
|---------|-----|---------|-----|
| Master节点 | 3台 | 8核CPU/32GB内存 | 集群管理、高可用 |

| Worker-网关节点 | 2台 | 8核CPU/32GB内存 | API网关、Web服务 |

### 9.2 核心服务容器配置

| 服务 | 副本数 | 资源配额(CPU/内存) |
