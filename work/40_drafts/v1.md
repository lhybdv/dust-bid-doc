# 1.1 项目背景

宁夏回族自治区位于西北生态安全屏障的关键区域，地处腾格里沙漠南缘，是我国沙尘天气高发区之一。沙尘天气对区域交通安全、人体健康、旅游产业和公众生产生活造成直接影响。近年来极端沙尘事件呈增多趋势，精细化预测和提前预警能力成为气象服务的刚性需求。

《宁夏沙尘预测AI模型训练及示范应用项目》正是在这一背景下提出的建设任务。本项目的核心目标是构建沙尘动态预测AI模型训练能力，并以此为基础建设沙尘监测预警示范服务场景应用平台，将预测结果转化为面向公众、行业系统和移动端的可使用、可联动服务。

从业务需求看，当前沙尘气象服务存在以下突出问题：

一是预测提前量不足。传统数值预报模式对沙尘过程的预报提前时间有限，难以满足交通管理和公众出行提前决策的需求。本项目要求沙尘动态预测AI模型实现24小时以上预警提前量，为交通管控和公众防护争取更充裕的准备时间。

二是服务转化能力薄弱。沙尘预报结果目前停留在数值产品层面，未能有效转化为公众易理解的风险提示、行业系统可调用的接口、以及差异化场景的防护建议。本项目需要构建预测结果到场景服务的完整转化链路，覆盖交通、人体健康、旅游和公众四个场景。

三是平台支撑能力不足。现有系统难以支撑7×24小时不间断运行、100用户50并发访问以及秒级界面响应。平台需要采用微服务架构和流批一体化框架，在性能、稳定性和可扩展性上满足业务要求。

本项目面向宁夏回族自治区气象局履约，合同签订后24个月内完成验收。项目建设将填补宁夏区域沙尘AI精细化预测的业务空白，形成"模型训练—监测预警—场景服务—行业联动"的完整能力链条，提升区域气象服务的专业化、智能化水平。

--># 1.2 建设目标

本项目的建设目标是构建覆盖沙尘预测、监测预警和场景服务的完整能力体系，形成"预测能力输出—监测预警展示—场景服务转化—行业系统联动"的一体化解决方案。

**目标一：沙尘动态预测AI模型训练与检验评估。** 基于物理机理与深度学习融合方法，构建适用于宁夏区域的沙尘动态预测AI模型，实现预警提前24小时以上、沙尘浓度预测准确度75%以上的业务化预报能力，并通过历史个例回溯和实况监测数据开展模型检验评估，形成持续迭代优化机制。

**目标二：沙尘监测预警示范服务场景应用平台。** 建设面向专业气象人员和行业用户的沙尘监测预警示范服务场景应用平台，集成沙尘实况监测、预报预警、产品制作和GIS空间分析功能，支持7×24小时不间断运行，界面响应不超过3秒、前端交互不超过2秒，满足100用户50并发的性能要求。平台采用微服务架构和流批一体化框架，具备可扩展和易维护特性。

**目标三：三领域智能体服务接入。** 通过标准化API网关接入宁夏移动建设的交通、人体健康、旅游三领域智能体，完成参数组织、调用编排、结果解析和融合展示，向行业系统提供结构化的场景风险查询和预警订阅服务。本项目不重复建设三领域智能体，专注于接口适配和服务融合。

**目标四：微信小程序公众服务。** 开发面向公众的微信小程序，提供实时能见度地图、当前位置沙尘风险等级、未来趋势和预警提醒、以及个性化防护建议，并支持关注区域订阅和消息推送功能。

四个目标形成完整的能力链条：模型训练提供核心预测能力，平台提供业务支撑环境，智能体和小程序将预测结果转化为差异化场景服务，最终向公众和行业系统输出可使用、可联动的沙尘气象服务产品。

# 1.3 建设内容

本项目建设内容包括沙尘动态预测AI模型训练、沙尘监测预警示范服务场景应用平台、三领域智能体服务接入和微信小程序四个部分。

**沙尘动态预测AI模型训练。** 构建适用于宁夏区域的沙尘动态预测AI模型，融合物理机理与深度学习方法，基于历史沙尘天气过程、气象观测、数值模式、卫星遥感和大气环境监测等多源数据建立训练数据集，实现未来24小时以上沙尘浓度或风险空间分布的精细化预报。配套建立模型检验评估流程，通过历史个例回溯、实况监测和遥感判识对模型性能进行持续评估。

**沙尘监测预警示范服务场景应用平台。** 建设面向专业用户的B/S架构平台，集成沙尘实况监测、预报预警、产品制作、GIS空间分析和系统管理功能。平台采用微服务架构和流批一体化框架，支持7×24小时不间断运行，界面响应不超过3秒、前端交互不超过2秒，满足100用户50并发的性能要求，具备可扩展和易维护特性。

**三领域智能体服务接入。** 通过标准化API网关接入宁夏移动建设的交通、人体健康、旅游三领域智能体，完成参数组织、调用编排、结果解析和融合展示。本项目负责接口适配和服务融合，不重复建设三领域智能体。行业系统可通过API接口查询区域风险、道路通行风险、景区影响等级，并订阅预警信息。

**微信小程序。** 开发面向公众的微信小程序，提供实时能见度地图、当前位置沙尘风险等级、未来趋势和预警提醒、个性化防护建议，以及关注区域订阅和消息推送功能。

上述四部分内容相互支撑：模型训练提供核心预测能力，平台提供业务运营环境，智能体接入扩展场景分析深度，小程序触达公众用户。项目整体在合同签订后24个月内完成验收。

# 1.4 重点难点分析

本项目的技术重点和难点主要体现在模型训练、平台性能和场景服务转化三个方面。

**重点一：物理机理与深度学习融合的沙尘预测模型。** 沙尘预报涉及起沙、传输和沉降的复杂大气物理过程，纯数据驱动的深度学习模型难以充分表达物理规律。本项目采用物理机理与深度学习融合方法，需要在特征构建阶段引入大气动力学约束，在训练阶段保留物理一致性，在预测阶段输出符合大气运动规律的沙尘浓度空间分布。这一融合路线的实现难度在于：如何提取适用于宁夏腾格里沙漠南缘局地起沙特征、如何构建迁移学习或少样本增强机制以适应有限的历史沙尘个例、如何保证模型在业务条件下的稳定性和可解释性。

**重点二：高并发低延迟的平台服务能力。** 采购要求平台支持7×24小时不间断运行、100用户50并发、界面响应不超过3秒、前端交互不超过2秒。现有气象服务平台在并发能力和响应速度上普遍存在瓶颈，本项目需要通过微服务架构拆分、缓存策略优化、异步处理机制和流批一体框架来解决高并发低延迟问题。此外，平台需要同时支撑实况监测、预报预警和场景服务三类业务，对系统资源调度和优先级处理提出更高要求。

**重点三：多源数据的融合处理与质量控制。** 沙尘预测模型训练依赖气象观测、数值模式、卫星遥感、地表状况和大气环境监测等多源异构数据。不同来源数据的时空分辨率、精度、时效性和更新频率各不相同，数据质量参差不齐。本项目需要建立标准化的数据接入、清洗、质量控制和融合处理流程，确保训练数据的一致性和可用性。多模态数据融合处理本身也是评分细则明确要求的考核点。

**重点四：场景服务的智能转化与行业联动。** 预测结果向场景风险等级、公众提示和行业接口的转化涉及复杂的业务逻辑和参数映射。三领域智能体由宁夏移动建设，本项目需要完成接口适配和调用编排，在没有智能体详细接口文档的情况下保证服务融合的可靠性。行业API的标准化、限流和异常处理也需要在设计中充分考虑。

上述四个重点与后续技术方案章节形成对应：模型融合问题对应3.1节，平台性能问题对应2.5节和3.3节，数据融合问题对应3.1.1节，场景转化问题对应3.2节和3.3.3节。

# 2 总体技术方案

## 2.1 总体架构设计

### 2.1.1 架构设计目标与原则

本项目以沙尘动态预测AI模型训练为基础，以沙尘监测预警示范服务场景应用平台为载体，构建覆盖公众、交通、人体健康、旅游等场景的沙尘影响智能评估与服务转化能力。方案完整响应采购文件关于预测提前量、准确度和平台性能的硬性要求，并突出预测结果向风险等级、行业提示、公众建议和联动接口转化的应用价值。

本项目采用微服务架构、流批一体化框架和B/S架构模式，整体系统划分为模型训练层、平台服务层和场景服务转化层三层结构，各层职责明确，层间通过标准化接口通信。

### 2.1.2 整体架构

系统整体采用微服务架构，按业务边界拆分为独立部署的服务单元，各服务可通过统一API网关对外暴露。平台服务端采用B/S架构，浏览器侧为Web应用，移动侧为微信小程序，外部系统通过RESTful API接入。

## 2.2 技术架构与技术路线

### 2.2.1 技术架构选型

本项目以微服务架构、流批一体化框架和B/S架构为核心，构建覆盖模型训练、平台服务和场景输出的完整技术底座。三种架构模式相互配合，支撑沙尘预测AI模型从训练到推理的全链路能力。

**微服务架构**按业务边界将系统拆分为独立服务单元，各服务独立部署、独立扩展，通过服务注册与发现机制实现协调。本项目将预测服务、实况监测服务、预警服务、场景服务、GIS服务、接口服务等拆分为独立微服务，各服务使用容器化部署，支撑动态扩缩容，响应REQ-S4-01（微服务架构）。

**流批一体化框架**将实时数据处理与离线批处理统一管理。气象站、监测站、能见度等实时数据通过流式管道处理，模型推理的批量预测和历史回溯通过批处理管道执行，两套管道共享特征存储和模型存储层，统一由流批一体处理引擎调度，响应REQ-S4-02（流批一体化框架）。

**B/S架构**采用浏览器/服务器模式，用户通过Web管理端和微信小程序访问平台服务。服务端提供JSON over HTTPS标准API，支撑跨平台、跨终端的灵活接入，响应REQ-S4-03（B/S架构）。

### 2.2.2 技术路线设计

本项目技术路线围绕"物理机理+深度学习融合"这一核心方法展开，分为数据融合处理路线、模型训练路线和平台服务路线三条主线。

**数据融合处理路线**：多源数据经质量控制后进入融合处理管道。气象观测数据（温度、湿度、风速、风向、气压）作为基础输入，数值模式产品（ECMWF、GFS）提供大尺度背景场，卫星遥感数据（风云FYD、MODIS）提供空间覆盖，地表和大气监测数据补充地面实况。各源数据经标准化、质量控制、时间对齐和空间插值后，在特征层面进行融合，生成模型输入特征张量。融合过程采用时空插值和特征交叉方法，保留各源数据的优势信息。

**模型训练路线**：融合特征进入沙尘动态预测AI模型训练管道。本模型采用物理机理引导的深度学习架构，底层网络为时序神经网络（长短期记忆网络LSTM或时序卷积网络TCN），在输入层和损失函数中嵌入沙尘传输的物理约束（如风场驱动、扩散方程）。训练过程使用历史个例数据（宁夏及周边地区历史沙尘个例）进行监督学习，采用迁移学习方法利用少量样本完成模型适配。训练完成后模型以服务化方式部署，接收实时融合特征并输出未来24小时以上沙尘风险空间分布（网格分辨率待确认）和浓度等级。

**平台服务路线**：模型推理结果推送到平台服务层后，分三路处理——实时预警判断（流式管道）、历史回溯分析（批式管道）和空间制图（GIS管道）。预警判断根据风险阈值生成预警信号并触发发布流程；历史回溯支撑模型效果评估和个例复盘；空间制图生成可视化产品供Web端和小程序展示。三路结果统一写入结果存储层，支撑多渠道调用。

### 2.2.3 技术栈组合

本项目主要技术栈选型如下：

| 技术层级 | 选型 | 说明 |

### 2.2.5 关键技术难点与应对

本项目技术路线面临以下关键难点：

**物理机理与深度学习融合**：单一深度学习模型在沙尘预报中易出现过拟合和泛化能力不足问题。本项目采用物理机理引导的损失函数设计，在常规交叉熵或均方误差损失基础上增加风场驱动约束和连续性约束，使模型在物理上更合理。应对措施为在模型训练前开展宁夏历史沙尘个例的物理规律分析，并据此设计约束项。

**小样本模型训练**：宁夏地区沙尘天气个例有限，纯数据驱动方法面临样本不足问题。本项目采用迁移学习方法，利用已有全国或更大范围的沙尘模型权重进行微调（fine-tuning），结合数据增强（时间序列增强、空间扰动）扩大训练样本集。

**流批一体调度复杂性**：实时预警和离线分析共用同一套处理引擎，增加了调度复杂性。本项目采用Flink作为统一引擎，通过作业链（Job Graph）管理流式和批式任务的依赖关系，设置独立的作业监控和异常告警。
