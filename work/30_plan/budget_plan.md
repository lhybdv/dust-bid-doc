## 内容投资组合

### 投资逻辑

本项目技术投标书以 **"模型训练+示范应用"** 为主线，46分技术方案为核心驱动力。全书按以下优先级分配篇幅：

1. **硬响应**：S0/S1/S2 实质性要求必须在对应章节中完整覆盖，不得遗漏
2. **高分评分项**：46分技术方案中，模型训练（18分）和平台建设（18分）合计36分，需重点展开
3. **交付可信度**：实施、测试、验收和售后章节让评审专家相信方案可落地

### 内容块划分

| 内容块 | 页数范围 | 评分分值 | 说明 |
|--------|---------|---------|------|
| 需求理解 | 8~10页 | 10分 | 项目背景、目标、内容、重点；用户资料充足 |
| 总体技术方案 | 20~24页 | 10分 | 架构+接口+部署+非功能；需配多个技术图 |
| 模型训练与验证 | 22~26页 | 18分 | **核心差异化**，数据/方法/训练/验证/输出/验收 |
| 平台建设 | 20~24页 | 18分 | 功能架构+监测预警+场景服务+GIS联动 |
| 微信小程序 | 3~4页 | 0 | 轻量接入，不单独计分 |
| 实施管理 | 10~12页 | 5分 | 开发计划+测试方案+质保体系；配甘特图 |
| 培训方案 | 4~5页 | 5分 | 响应2次以上培训要求 |
| 售后服务 | 5~6页 | 5分 | 响应7*24运维要求 |
| 综合评价 | 3~4页 | 5分 | 总结亮点，不宜过长 |
| **合计** | **95~115页** | **76分** | |

### 评分权重与页数分配对应

```
46分技术方案（最高）  →  62~74页（占65%~68%）
10分需求理解          →  8~10页（占8%~9%）
5分 实施管理          →  10~12页（占10%~11%）
5分 培训              →  4~5页（占4%~5%）
5分 售后              →  5~6页（占5%~6%）
5分 综合评价          →  3~4页（占3%~4%）
```

### 模板约束分析

模板正文骨架为"投标人自行编写的技术方案"，技术部分正文从第1章开始。

- **正文主战场**：第1~7章（对应内容块）
- **非正文部分**：封面、目录、评分索引由构建工具自动生成，不占用手动写作页数
- **图表策略**：每个主要技术节（2.1、2.2、3.1.1、3.1.2、3.3.1、4.1）均配一张技术图，图表计入正文页数但不占文字空间
